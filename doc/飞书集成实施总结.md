## 🚀 使用说明

### 1. 配置飞书应用

#### 1.1 在飞书开放平台创建应用
1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 `App ID` 和 `App Secret`

#### 1.2 配置权限
在飞书开放平台应用管理中添加以下权限：
- `im:message` - 接收和发送消息
- `im:resource` - 下载文件

#### 1.3 启用机器人
在应用中添加机器人能力

### 2. 配置环境变量

复制 `config/.env.example` 到 `.env`，添加飞书配置：

```env
# 启用飞书
FEISHU_ENABLED=true

# 飞书应用凭证
FEISHU_APP_ID=cli_xxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxx

# 飞书机器人名称
FEISHU_BOT_NAME=NoteAgents

# 授权用户列表（从日志中获取）
FEISHU_ALLOWED_USER_IDS=ou_xxxxxxxxxxxxx

# 轮询模式设置
FEISHU_POLL_INTERVAL=1.0

# Webhook模式（可选，需要公网IP）
FEISHU_USE_WEBHOOK=false
FEISHU_WEBHOOK_HOST=0.0.0.0
FEISHU_WEBHOOK_PORT=8000
```

### 3. 运行程序

#### 使用多平台版本（推荐用于飞书）
```bash
# Windows
.venv\Scripts\python.exe main_multi.py

# Linux/Mac
source .venv/bin/activate
python main_multi.py
```

#### 使用原 Telegram 版本（保持不变）
```bash
python main.py
```

### 4. 获取用户 ID

1. 启动程序后，给飞书机器人发送任意消息
2. 查看日志中的 `user_id`（格式：`ou_` 开头）
3. 将该 ID 添加到 `FEISHU_ALLOWED_USER_IDS`

## 📖 架构说明

### 多平台架构设计
```
┌─────────────────────────────────────────────────────────┐
│                    User Interface                       │
│  ┌───────────────┐         ┌───────────────────────┐   │
│  │   Telegram    │         │        Feishu         │   │
│  └───────────────┘         └───────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  Access Layer                          │
│  ┌──────────────────┐  ┌──────────────────────────┐   │
│  │ Telegram Adapter │  │   Feishu Adapter         │   │
│  └──────────────────┘  └──────────────────────────┘   │
│  ┌──────────────────────────────────────────────────┐ │
│  │          Unified Message Format                  │ │
│  └──────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Message Coordinator                        │
│        (统一消息处理逻辑，复用业务层)                │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Business Logic Layer                       │
│  (AI 总结、Obsidian 入库、GitHub 同步等 - 完全复用)   │
└─────────────────────────────────────────────────────────┘
```

### 核心模块

#### 1. `platforms.platform_types`
- `PlatformType` - 平台枚举（TELEGRAM, FEISHU）
- `ContentType` - 内容类型枚举
- `UnifiedMessage` - 统一消息格式
- `FileInfo` - 文件信息
- `FeishuPlatformConfig` - 飞书配置

#### 2. `platforms.base`
- `PlatformAdapter` - 适配器抽象基类
- 定义所有平台必须实现的接口

#### 3. `platforms.manager`
- `PlatformManager` - 平台管理器
- 管理多个平台适配器
- 消息路由和分发

#### 4. `platforms.feishu_adapter`
- `FeishuAdapter` - 飞书平台适配器
- 处理飞书 API 调用
- 消息发送/接收
- 文件下载

#### 5. `platforms.coordinator`
- `MessageCoordinator` - 消息协调器
- 统一处理来自不同平台的消息
- 复用业务逻辑层

## 🔧 设计优势

### 1. 向后兼容
- 原有的 `main.py` 完全保留
- Telegram 用户不受影响
- 可逐步迁移到多平台版本

### 2. 架构清晰
- 平台层与业务层分离
- 适配器模式易于扩展新平台
- 统一消息格式简化处理

### 3. 业务复用
- AI 总结、Obsidian 入库等核心逻辑完全复用
- 不需要为飞书重写业务代码
- 测试和维护成本降低

### 4. 灵活配置
- 可独立启用/禁用各平台
- 权限独立控制
- 支持轮询和 Webhook 两种模式
